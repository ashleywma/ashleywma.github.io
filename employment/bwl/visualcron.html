VisualCron applications (internal)

Portfolio summary only — the running app and repository stay private to the org.

Purpose

Internal directory for active scheduler jobs: one web page that answers “what runs, how often, when it last ran, and who gets notified on success or failure”—so people do not have to piece that together only from the native scheduler client.

Stack

ASP.NET Core 8, Razor Pages, single main page plus a download handler. Dependency injection registers a scoped repository that loads the job list on each GET.

Data

Jobs live in SQL Server. The app reads a relational catalog and keeps only enabled rows. Access goes through ADO.NET (SqlConnection, SqlCommand, async reader) into a small C# row model—no ORM.

The connection string is stored encrypted in configuration and decrypted when the process starts, so credentials are not committed as plaintext.

UI

Results render as an HTML table (name, description, cadence, last run, email targets for success and failure). DataTables adds client-side paging and sorting on that table after load.

Export

A Razor page handler builds the same columns in an Excel workbook via ClosedXML and returns a download with a timestamped filename (audits, handoffs, or attachments).

Hosting & access

Standard ASP.NET Core pipeline: HTTPS redirection, static files, Razor routing. Production can enforce an enterprise authorization policy (e.g. directory group); dev often relaxes that for local work. Deployed behind IIS on the org network. Deployment stays on the org network—not exposed through this personal site.

Skills

Backend

.NET 8, ASP.NET Core, Razor Pages, dependency injection, async/await, ADO.NET

Data

SQL Server, SqlConnection, SqlCommand, async data readers, relational catalog modeling

Frontend

HTML/CSS, DataTables, JavaScript, responsive tables, client-side sorting & paging

Export & config

ClosedXML, Excel (.xlsx) export, encrypted configuration, IIS deployment, internal networking
