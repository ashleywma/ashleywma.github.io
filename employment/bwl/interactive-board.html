Water Department Board (interactive)

Portfolio summary only — the deployed device/app and any partner assets stay private.

What it is

An interactive learning device designed to help children learn about water conservation. The device combines physical inputs (buttons / sensors via a microcontroller) with a guided on-screen experience.

My role

I built the software portion: the application UI, state transitions, and hardware input integration that drives the activity flow.

The hardware portion was built by Meng Shi; we collaborated closely to align device inputs with the on-screen experience.

Stack

Prototype implementation used .NET with a component UI and a lightweight service that listens for serial input from the device controller (e.g. Arduino).

Hardware integration

The app listens on a configured serial port, parses button messages, and maps them to UI events so the physical board can navigate steps, confirm choices, and advance the experience without a keyboard/mouse.

Skills

Frontend

.NET, Blazor Server, MudBlazor, component UI, state management

Hardware I/O

Serial I/O, System.IO.Ports, event-driven input, microcontroller integration

Engineering

Configuration, environment-specific settings, error handling, device availability checks
